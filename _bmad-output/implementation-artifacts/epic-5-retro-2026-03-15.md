## What Needs Improvement

### 1. Story Planning Missed Global Prerequisites (CRITICAL)

**Incident:** Epic 5 original plan (5-1 to 5-8) did not include a global navigation shell story. Dev Agent and SM both failed to identify this prerequisite. Only when Alexyu ran local dev server did the team discover the UI was completely wrong — no header, no tabs, no dark theme baseline.

**Root Cause:**
- create-story workflow does not enforce a "global prerequisite check" against design screenshots
- Dev Agent did not run local dev server before starting 5-1 to verify baseline UI state
- SM did not cross-reference design screenshots against story list during sprint planning

**Impact:** Story 5-0 had to be created mid-sprint, causing rework. All subsequent stories built on a foundation that wasn't originally planned.

### 2. UX Design Verification Not Enforced (CRITICAL)

**Incident:** Despite modifying the dev-story workflow to require design verification, the process was not correctly executed. Stories 5-2 through 5-7 all had design deviations that were not caught until:
- Story 5-9 remediated deviations from 5-2 through 5-5
- Story 5-10 remediated deviations from 5-6 and 5-7

**Root Cause:**
- Design verification was added as guidance, not as a mandatory task with checkbox
- Dev Agent could skip verification without being blocked
- SM and UX Designer validation was not triggered automatically
- No three-gate verification: Dev → SM → UX → User

**Impact:** 2 additional stories (5-9, 5-10) were required to fix accumulated design deviations. This represents ~18% of epic effort spent on remediation.

### 3. Initial Implementation Quality Still Low (RECURRING — Epic 3, 4, 5)

**Pattern:** 60+ code review issues across 11 stories. This is the third consecutive epic with this problem.

| Recurring Pattern | Stories Affected | Examples |
|-------------------|-----------------|----------|
| Feature built but not wired | 4/11 | 5-1: PosterCardMenu/SettingsGear not connected to Grid; 5-5: useLibraryStats defined but never consumed |
| AC requirements not completed but marked done | 3/11 | 5-3: search highlighting not implemented; 5-7: Shift+Click not done; 5-8: fade-in animation missing |
| listAll pagination/sort bugs | 2/11 | 5-5: page>1 broken; 5-8: merge-sort missing |
| Backend column mapping errors | 2/11 | 5-2: release_date/rating mismatch; 5-4: vote_average mapping |
| Navigation params dropped | 2/11 | 5-2: sort/view params lost on navigate |

**Root Cause:** Dev Agent does not perform systematic self-verification before declaring story complete. Code review remains the sole quality gate.

### 4. Tailwind v3 vs v4 Compatibility Issue

**Incident:** Tailwind CSS version differences caused styles not to be applied to elements correctly. Dev Agent wrote syntactically correct classes that didn't render in the browser.

**Root Cause:** project-context.md did not document Tailwind version or known breaking changes. Unit tests don't catch CSS rendering issues.

### 5. CI Pre-Commit Hook Incomplete Coverage

**Incident:** Despite husky + lint-staged being installed (Epic 4 Action Item #1), CI still failed during 5-1 development. See: https://github.com/j620656786206/vido/actions/runs/22754129464

**Root Cause:** Action item completion was measured as "tool installed" rather than "problem eliminated." The hook may not cover all CI checks, or may have been bypassed.

### Action Item #1: Mandatory Design Verification Task in Every UI Story

**Problem:** UX design deviations accumulated across stories, requiring 2 remediation stories (5-9, 5-10).

**Solution:** Modify create-story workflow to automatically add a mandatory Design Verification task to every story that has UI components:

```
- [ ] Task N: Design Verification (MANDATORY — cannot mark story as done until complete)
  - [ ] N.1: Dev Agent compares running app against design screenshots, lists matches/deviations
  - [ ] N.2: SM (Bob) reviews Dev Agent's comparison report
  - [ ] N.3: UX Designer (Sally) verifies implementation against design
  - [ ] N.4: Verification results submitted to Alexyu for final approval
  - [ ] N.5: All deviations either fixed or explicitly waived by Alexyu
```

**Enforcement:** Task N incomplete → Story CANNOT be marked as done. Dev Agent, SM, and UX Designer must all sign off.

**Files to modify:**
- `_bmad/bmm/workflows/4-implementation/create-story/` — add design verification task generation
- `_bmad/bmm/workflows/4-implementation/dev-story/` — block completion if design verification task is not done
- `_bmad/bmm/agents/dev.md` — add design verification to agent instructions

### Action Item #2: Dev Agent Pre-Completion Self-Verification Checklist

**Problem:** Features built but not wired, AC not completed but marked done.

**Solution:** Add mandatory self-verification step to dev-story workflow, executed BEFORE story can transition to "review" status:

1. **AC Line-by-Line Verification** — Dev Agent must respond to each AC with: ✅ Done (file:line) or ❌ Not Done (reason)
2. **Wiring Verification** — Every new component must be verified as imported and rendered in a parent
3. **Visual Verification** — Dev Agent must confirm "I have verified the UI in browser" (not just tests)
4. **Test Verification** — All existing tests pass + new tests cover new functionality

**Files to modify:**
- `_bmad/bmm/workflows/4-implementation/dev-story/` — add pre-completion gate
