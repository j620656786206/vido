## What Went Well

1. **Interface decoupling consistency** — All 3 AI stories followed project-context plugin decoupling principles. TextCompleter interface (9-1), semaphore concurrency (9-2a), TranslationBlock type (9-2b) — clean architecture throughout.

2. **Pre-existing failure handling matured** — Epic 9c retro AI2 rule (fix-or-file) worked. 3 preexisting-fail stories went through full pipeline: story file → dev → CR → sprint-status tracking.

3. **Test coverage strong** — 128 new tests with thorough edge case coverage. SRT round-trip, partial failure fallback, multi-line subtitle handling all covered.

4. **Design-after-backend pipeline** — 9-UX design was more efficient because all 3 backend APIs were already implemented. Designer knew exact API contracts.

5. **Epic 8 retro follow-through 94%** — Sprint-status tracking rule directly drove accountability.

---

## Challenges

1. **Pre-existing test failures discovered too late** — 3 failures (library_service, setup_service, download_handler) were caused by earlier epics but only surfaced during Epic 9's `go test ./...`. No epic-level regression gate exists.

2. **Recurring security patterns** — dev agent repeatedly produced unbounded I/O and context lifecycle issues across 9-1, 9-2a, 9-2b. Code review caught them every time but the pattern keeps recurring.

3. **Package dependency boundaries undocumented** — subtitle↔engine circular import constraint was discovered during 9-2b implementation. Forced inline SRT parser instead of reusing subtitle.ParseSRT.

4. **UX story status ambiguity** — 9-UX story file says "ready-for-dev" but sprint-status says "done". UX design stories need a clearer status definition.

5. **Dead code accumulation** — DS, CR, and TA stages can all produce dead code (deleted callers but not functions, unused helpers). No automated detection in CI.

6. **Local vs CI lint gap** — Pre-commit hook disabled (Rule 12), local lint depends on IDE extensions (version drift). Developers don't see lint errors until CI fails.

---

## Key Insights

- **Tracking rules work** — The jump from 17% → 94% retro follow-through proves that putting items in sprint-status.yaml with agent routing creates accountability.
- **FFmpeg is container-agnostic** — Whisper pipeline works with any video format (MKV, AVI, MP4, etc.), not just MKV as story docs implied. No code changes needed.
- **Quality gate feedback loop matters** — The further the lint/test gate is from the developer (local → CI), the more wasted cycles on failed pushes.
