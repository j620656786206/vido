## Challenges (Patterns from Deep Story Analysis)

### 🔴 Pattern #1 — A11y (keyboard / focus / ARIA) keeps being caught at CR, not pre-flight

- 11-2: bottom sheet declared `role="dialog" aria-modal="true"` with **no keyboard affordances** → CR M2 added Escape-to-close + initial focus into the sheet.
- 11-3: combobox needed `aria-activedescendant` + listbox-only-options structure + person-row affordance → all CR fixes.
- This is a **repeat of the exact class Epic 10's AI-1 was meant to prevent**. Root cause above. **This is the #1 action item.**

### 🟠 Pattern #2 — TanStack Router numeric search-param coercion gotcha (recurred)

- 11-2: a single-value deep link (`?genre=16`/`?platform=8`) silently dropped the filter because TanStack Router JSON-parses a lone numeric query value into a `number`, failing the route's `typeof === 'string'` guard. Fixed via `toCsvString()` + defensive `String()` coercion, guarded by E2E.
- **Recurred** in Story 8-11 (`subtitleStatus` param). Two strikes → worth a project-context note before a third.

### 🟢 Pattern #3 — Minor File-List hygiene misses

- 11-4 M2 + 11-2: spec files committed but not listed in File List until CR flagged it. Low severity; the existing checkbox audit mostly catches it.

## Key Insights

1. **Enforcement mechanism predicts stickiness.** Action items that become a mandatory workflow action (with a recorded line) or an automated check (grep / ESLint) stick; passive checklists in a side document do not. Future retro action items should prefer the former.
2. **CR is doing heavy lifting that should shift left.** The same a11y + validation classes recur; moving them to lint/pre-flight frees CR for higher-order review.
3. **Framework gotchas recur across epics** (TanStack param typing) — codify once, in project-context, the moment a pattern hits twice.
