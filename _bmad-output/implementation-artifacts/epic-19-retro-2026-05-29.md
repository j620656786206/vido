## 1. Epic Summary

Epic 19 is the **systemic response to the `bugfix-10-4` root cause** (Party Mode 2026-05-08:
`HoverPreviewCard.tsx` had been independently invented and diverged from the `.pen`
`Component/PosterCardHover` node `MQbvp`, undetected for months). It is a cross-cutting,
non-functional drift-prevention initiative — Owner: Winston (rules) + Amelia (tooling/CI) +
Sally (sweep coordination).

### Delivery Metrics

- **Completion:** 13 / 13 stories done (100%)
  - Primary: 19-1, 19-2, 19-3, 19-4, 19-4b, 19-5, 19-6, 19-7, 19-8, 19-9
  - Bugfix follow-ups: bugfix-19-4b-1, bugfix-19-4b-1-followup, bugfix-19-9
- **Two structural halves:**
  - **Visual half:** 19-3 ESLint rule → 19-4/19-4b harness+baselines → 19-5 CI gate → 19-8 sweep → 19-9 time-dimension hardening
  - **Journey half:** 19-6 TestSprite monthly consumer → 19-7 month-end watchdog
- **Rules delivered:** 3 mandatory rules covering 3 orthogonal drift dimensions
  - **Rule 21** — spatial (code ↔ `.pen` node traceability)
  - **Rule 22** — cadence (per-epic-retro sampling audit)
  - **Rule 23** — temporal (clock-mocked fixture stability)
- **Tooling:** 2 custom ESLint rules + Playwright `visual` project + 3 GitHub Actions workflows
- **Audit conclusion:** 0 material drift / 2 minor across all 131 components (19-8 full sweep)

### Headline Finding

> **The central hypothesis was empirically DISPROVEN.** Epic 19 was launched on the belief
> that design-implementation drift was *systemic*. The 19-8 capstone full sweep of all 131
> components found **0 material drift, 2 minor (log-only)**. The `bugfix-10-4` drift was an
> **isolated incident, not a pattern**. The prevention toolchain is now in place regardless —
> drift detection has shifted from reactive (user reports a bug) to proactive (ESLint + CI gate).

## 2. What Went Well

1. **Being wrong, well.** Sally's "systemic drift" hypothesis was disproven by her own full
   sweep — a healthy, evidence-driven conclusion rather than a defended assumption.
2. **Clean three-dimensional model.** Rule 21 (spatial) / Rule 22 (cadence) / Rule 23 (temporal)
   each guard a distinct, orthogonal drift class; all enforced via `pnpm lint:all` ⇒ CI.
3. **Reusable infrastructure.** Plan-D single-fixture isolation (`?fixture=<id>`) solved the
   Radix-portal/`fixed inset-0` pointer-interception class in 19-4b and was **reused** as the
   bisection harness for bugfix-19-4b-1.
4. **Fresh-context Code Review caught real HIGH findings every time** — most notably 19-5 H2
   (a `paths:` filter listing two non-existent paths and missing the real `styles.css`, which
   would have silently bypassed the entire visual gate).
5. **Symptom→class→meta escalation done right.** The time-bomb chain was resolved by treating
   the *class* (Rule 23 + ESLint + `withFixedClock` + dual-state baselines + SM template field),
   not just reblessing the failing baseline. Winston: "治標+治本綁同一故事."
6. **Empirics overturned expert intuition.** The bugfix-19-4b-1 bisect proved all four of
   Sally's "prime suspects" produced 0 warnings in isolation; the real offender was
   `useParseProgress.ts` — the strongest validation of the BISECT-GATED story structure.

## 3. Challenges & Recurring Patterns

### ① "Silently bypass the gate" failure class (3+ stories)

Gate configuration with holes that read as green but don't actually guard:
- 19-5 H2 — `paths:` filter with two non-existent paths, missing real `styles.css`
- bugfix-19-4b-1-followup M1 — dorny filter omitted `pnpm-lock.yaml` (same gap noted in `visual-regression.yml`)
- bugfix-19-4b-1-followup H1 — Rule 20 ack wrapped in backticks defeated the grep helper

### ② Time/clock determinism — a half-learned lesson

19-4b learned "pin the date" (CountdownTimer 2099→2020) but **did not generalize it**. The same
class bit again at 19-8 PR #8 (`library-recently-added`), forcing Rule 23. A half-lesson is more
expensive than none — you believe it's solved.

### ③ "first-run-only" assumption written wrong twice

The Linux-baseline bootstrap "runs only on first execution" assumption appeared in both 19-5 and
19-9 Dev Notes, surfacing only when 19-9 became the first incremental addition → bugfix-19-9.

### ④ External-tool reality ≠ story assumptions (correct-course hot zone)

- **19-6 Phase 2:** the bare `@testsprite/testsprite-mcp` CLI has no `check_account_info`
  subcommand → live-credit budget check unimplementable; `credits_*` permanently null; 62→30 cases.
- **19-7 full re-architecture:** drafted against 19-6 `[@contract-v1]`, but 19-6's CR had already
  bumped to `v2` → entire mechanism re-architected (live-credit threshold → `last_run`-health
  watchdog) before a single line shipped (sprint-change-proposal 2026-05-20). v1 never shipped.

### ⑤ Contract text persistently lagging shipped behavior

Rule 20 `[@contract-vN]` bumps: 19-3 v1→v2, 19-5 v1→v2 (×3 ACs), 19-6 v1→v2, 19-8 v1→v2,
19-3 v2→v3 (via 19-8), 19-5 v2→v3 (via bugfix-19-9). Consistently the contract was *widened to
match reality* — self-healing, but CR was perpetually reconciling text, and the grep tooling
itself had a near-miss.

### Velocity / completion observations (no time estimates)

- 9 primary stories spawned 3 bugfix follow-ups; **every bugfix traces to a finding the
  originating story flagged but did not fully resolve** (the find→fix gap — see Action Item P1).
- **Time-bomb discovery chain (4 links):** 19-4b CountdownTimer (half-lesson) → 19-8 PR #8 CI red
  → 19-9 (Rule 23 class fix) → bugfix-19-9 (CI incremental-bootstrap gap exposed by 19-9's own merge).
- 2 stories needed full pre-implementation re-scopes (19-4, 19-7); 1 needed a post-merge
  re-architecture (19-5).

## 8. Key Takeaways

1. **Drift is not systemic** — the founding hypothesis was disproven; prevention infrastructure
   stands on its own merit, not on a crisis that didn't exist.
2. **Half-learned lessons are expensive** — pin-the-date (19-4b) wasn't generalized and recurred;
   generalize fixes into rules at the moment of the lesson, not after the second occurrence.
3. **The find→fix gap is the dominant velocity drag** — every bugfix this epic was a deferred
   discovery; Rule 24 (forced discovery triage) directly targets it.
4. **Contracts heal by widening to reality** — Rule 20 worked, but downstream drafts get stranded
   by silent bumps (19-6→19-7); P2 closes that notification gap.
