## What Could Be Improved

### 1. Formatting/Lint Not Caught Pre-Commit
CI failed on Prettier formatting. The `.husky/pre-commit` hook isn't executable, so formatting issues slip through to CI. This wasted a CI cycle.

**Action Item:** Fix husky pre-commit hook permissions (`chmod +x .husky/pre-commit`) or add format-on-save to editor config.

### 2. SSE Event Schema Gap
The `ScanProgressEvent` type in the spec had `filesFound` (camelCase) but the actual backend sends `files_found` (snake_case). The story spec also lists `files_parsed` and `files_matched` fields that don't exist in the backend yet. This caused confusion in implementation and required the `filesProcessed` estimation workaround.

**Action Item:** When writing Story specs that reference API schemas, always verify field names against the actual Go struct JSON tags.

### 3. Duplicated Test Fixtures
ScanProgressCard and ScanProgressSheet tests both define identical `baseScanningState`, `completeState`, and `cancelledState` objects. This was flagged in TA review but deferred.

**Action Item:** Extract shared test fixtures to `__tests__/fixtures.ts` when more scanner tests accumulate.

### 4. No Story for Backend SSE Events
The backend sends `scan_progress` events, but there's no `scan_complete` or `scan_cancelled` event type defined in `hub.go`. Story 7-4 frontend listens for these events, but the backend doesn't emit them yet. The completion is detected via polling (STATUS_UPDATE when `is_scanning` flips to false).

**Action Item:** Add `EventScanComplete` and `EventScanCancelled` event types to `sse/hub.go` constants and emit them from the scanner service. This should be a tech debt story in Epic 8.

## Key Technical Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| useReducer for SSE state (not TanStack Query) | SSE is push-based, not request-response | Clean state machine, easy to test |
| Polling fallback on SSE error | Browser EventSource reconnects, but we need faster recovery | Works well, reconnects after 10s |
| 768px breakpoint for card vs sheet | Matches existing responsive patterns | Consistent with app shell |
| Auto-dismiss with shrinking progress bar | UX spec requirement, 10s timer | Smooth animation, pausable on hover |
| filesProcessed estimated from percentDone | Backend SSE doesn't send this field | Acceptable approximation |
